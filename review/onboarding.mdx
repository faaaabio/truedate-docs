## AP-OBR-004 — Date picker sem input manual e ano inicial incorreto

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Tela | Suas preferências (Data de Nascimento) |
| Responsável | — |

**Descrição:**
O campo "Data de Nascimento" depende exclusivamente do date picker nativo para preenchimento. Dois problemas identificados:
1. O usuário não consegue digitar a data manualmente, o que é mais rápido e acessível.
2. O calendário do date picker abre no ano atual (2026), dificultando muito a navegação para anos anteriores como 1989 — o usuário precisa navegar mês a mês.

**Correção esperada:**
- Permitir preenchimento manual da data diretamente no campo de input (formato `DD/MM/AAAA`).
- Adicionar ícone de calendário dentro do input para abrir o date picker como alternativa.
- Substituir o date picker atual por um seletor de mês/ano em colunas (scroll picker) para navegação mais rápida até anos como 1989.

## AP-OBR-005 — Títulos e subtítulos fora da fonte padrão na tela "Tudo Pronto!"

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Média |
| Tela | Tudo Pronto! (última etapa do onboarding) |
| Responsável | — |

**Descrição:**
Na tela de confirmação "Tudo Pronto!", o título principal e os itens da lista ("Perfis Verificados", "Algoritmo Inteligente", "Segurança em Primeiro Lugar") não estão aplicando a fonte padrão **Plus Jakarta Sans**. A fonte de fallback do sistema está sendo renderizada no lugar.

**Correção esperada:**
Verificar o carregamento da fonte `Plus Jakarta Sans` nessa tela específica e garantir que o token tipográfico correto está sendo aplicado ao título e ao componente de lista de benefícios.

## AP-OBR-006 — Botão "Finalizar" sobreposto por componentes

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Tela | Tudo Pronto! (última etapa do onboarding) |
| Responsável | — |

**Descrição:**
O botão "Finalizar" na tela "Tudo Pronto!" não é flutuante, fazendo com que elementos da lista de benefícios possam sobrepô-lo em dispositivos menores ou quando o teclado está ativo.

**Correção esperada:**
Tornar o botão flutuante com posicionamento fixo na parte inferior da tela (`position: fixed`, `bottom: safe-area-inset-bottom + 16px`), garantindo que nenhum componente de conteúdo o sobreponha.

## AP-OBR-008 — Botão "Continuar" sobreposto ao navbar do OS

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Tela | Criar sua conta (Cadastro) |
| Responsável | — |

**Descrição:**
O botão "Continuar" na tela de cadastro não respeita a safe area inferior do Android, ficando parcialmente sobreposto pela barra de navegação do sistema operacional quando o teclado está fechado. Em dispositivos com gesture navigation (sem botões físicos), o botão pode ficar inacessível.

**Correção esperada:**
Adicionar `paddingBottom` dinâmico baseado no `safeAreaInsets.bottom` do dispositivo. Em Android com gesture navigation, o valor mínimo deve ser `16px + altura do gesture bar`.
