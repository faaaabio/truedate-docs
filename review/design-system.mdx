## Tipografia

### AP-DS-001 — Fonte Plus Jakarta Sans não aplicada em múltiplas telas

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Telas afetadas | Onboarding, Cadastro, Tudo Pronto!, Perfil público, Configurações, Estatísticas, Filtros, Comunidades, Álbum, Mensagens, modais de Recurso Premium e Enviar Mensagem |
| Responsável | — |

**Descrição:**
A fonte padrão do app **Plus Jakarta Sans** não está sendo carregada ou aplicada consistentemente em diversas telas. Em vez dela, o sistema operacional renderiza sua fonte de fallback padrão (Roboto no Android, San Francisco no iOS), criando inconsistência visual generalizada em títulos, labels e textos de corpo.

O problema foi identificado nas seguintes telas (não exaustivo):
- Tela "Tudo Pronto!" — título e lista de benefícios
- Tela "Criar sua conta" — título e labels de seção
- Perfil público — seção "Informações Básicas"
- Configurações — nome do usuário, labels do menu
- Estatísticas — labels dos cards
- Filtros — título e opções
- Comunidades — título e tabs ("Todas", "Recentes", "Não lidas")
- Álbum — estado vazio ("Seu álbum está vazio")
- Mensagens — estado vazio ("Nenhuma conversa ainda")
- Modais — títulos de "Recurso Premium" e "Enviar Mensagem"

**Correção esperada:**
1. Verificar se a fonte `Plus Jakarta Sans` está sendo carregada via `expo-font` (ou equivalente) antes do primeiro render da aplicação.
2. Garantir que o carregamento é aguardado com `useFonts` + `SplashScreen.preventAutoHideAsync()`.
3. Confirmar que o `fontFamily: 'PlusJakartaSans'` está definido no tema global e que todos os componentes de texto (`Text`, `Heading`, labels) herdam corretamente.
4. Testar em dispositivos Android físicos — o problema pode não ser reproduzível em emulador.

## Cores

### AP-DS-002 — Cor da marca incorreta em múltiplas telas

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Telas afetadas | Tela de splash, header logo, elementos brand em geral |
| Responsável | — |

**Descrição:**
A cor da marca (`brand`) está sendo aplicada com um valor incorreto em algumas telas. A **cor correta da marca é `#C8B776`** (dourado/champagne). Em algumas telas foi identificado uso de cor diferente para elementos que deveriam usar a cor brand.

**Correção esperada:**
Atualizar o token de cor no design system / tema global:
```
brand.primary: #C8B776
```
Auditar todos os componentes que referenciam a cor brand e garantir que estão usando o token correto (não valores hardcoded).

### AP-DS-003 — Cor azul incorreta em elementos secundários

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Telas afetadas | Botões de ação, links, elementos de destaque em azul |
| Responsável | — |

**Descrição:**
Elementos que devem usar a cor azul (botões, links, indicadores) estão com valores de cor incorretos. As cores corretas definidas pelo designer são:

- **Azul primário:** `#0583F1` (substituir uso de preto `#000000` em contextos de ação)
- **Azul escuro / secundário:** `#143341` (usar como alternativa ao preto puro em textos e ícones)

**Correção esperada:**
Atualizar os tokens de cor no tema global:
```
color.blue.primary:   #0583F1
color.blue.dark:      #143341
```
Substituir usos de `#000000` em botões de ação e elementos de navegação pelos tokens corretos.

## Espaçamento

### AP-DS-004 — Espaçamento entre cards não segue a grade de 4px

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Média |
| Telas afetadas | Feed em grid, Estatísticas, Comunidades, Cadastro |
| Responsável | — |

**Descrição:**
O espaçamento entre cards e elementos em múltiplas telas não está seguindo a grade de espaçamento definida no design system: **todos os espaços devem ser múltiplos de 4px** (4, 8, 12, 16, 20, 24, 32...). Valores arbitrários como `10px`, `14px` ou `18px` foram identificados.

**Correção esperada:**
- Auditar todos os `gap`, `margin` e `padding` do app e arredondar para o múltiplo de 4px mais próximo.
- Definir um conjunto fixo de tokens de espaçamento no tema: `spacing.1=4`, `spacing.2=8`, `spacing.3=12`, `spacing.4=16`, etc.
- Priorizar as telas: Feed em grid (AP-DES-005), Cadastro e Estatísticas.

## Componentes

### AP-DS-005 — Drawer como componente padrão para ações secundárias

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Média |
| Telas afetadas | Modal "Recurso Premium", Modal "Enviar Mensagem", e outros modais do app |
| Responsável | — |

**Descrição:**
Atualmente, mensagens secundárias e ações de confirmação são exibidas como modais centrados (dialogs). O designer sugere migrar para o padrão de **bottom drawer** (sheet deslizável de baixo para cima), que é mais natural no mobile e deixa o app mais leve visualmente.

Telas identificadas com este padrão de dialog que poderiam usar drawer:
- "Recurso Premium" — exibido como modal centrado com botões "Assinar Premium" e "Cancelar"
- "Enviar Mensagem" — aviso de conduta exibido como modal com checkbox "Não mostrar novamente"

Adicionalmente, os **títulos dentro dos modais** também estão sem a fonte padrão (Plus Jakarta Sans), problema relacionado ao AP-DS-001.

**Correção esperada:**
- Criar um componente `BottomDrawer` reutilizável para uso em todo o app.
- Migrar os modais identificados para o novo componente.
- Garantir que o drawer respeita a safe area inferior e tem animação de entrada/saída suave.

### AP-DS-006 — Botões de ação primária devem ser flutuantes

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Telas afetadas | Onboarding, Cadastro, e demais telas com botão de ação principal na base |
| Responsável | — |

**Descrição:**
Botões de ação primária no rodapé de telas (ex: "Continuar", "Finalizar", "Próximo", "Buscar") não são flutuantes, permitindo que elementos de conteúdo os sobreponham ao rolar a tela ou quando o teclado está ativo. Em dispositivos com gesture navigation, o botão pode ainda ser parcialmente coberto pela barra do OS.

**Correção esperada:**
- Tornar estes botões flutuantes: `position: absolute`, `bottom: 0`, com padding inferior de `safeAreaInsets.bottom + 16px`.
- Criar um componente `FixedActionButton` padronizado para uso em todo o app.
