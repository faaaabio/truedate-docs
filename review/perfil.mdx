# Perfil — Apontamentos de Design

Telas cobertas: Adicionar Foto, Álbum, Perfil público, Configurações, Estatísticas.

---

## AP-PRF-004 — Erro ao adicionar foto pela câmera

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Tela | Adicione sua Foto (Onboarding) |
| Responsável | — |

**Descrição:**
Ao tocar no botão "Camera" na tela de adição de foto durante o onboarding, o app exibe um dialog de erro: **"Falha ao selecionar imagem. Tente novamente."** A câmera não abre e nenhuma foto é capturada, mesmo com permissões concedidas.

**Correção esperada:**
Investigar o handler de abertura da câmera nativa. Verificar se o `react-native-image-picker` (ou biblioteca equivalente) está configurado corretamente para câmera no Android. Checar permissões em runtime (`CAMERA` e `WRITE_EXTERNAL_STORAGE`).

## AP-PRF-005 — Erro ao adicionar foto pela galeria (onboarding)

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Tela | Adicione sua Foto (Onboarding) |
| Responsável | — |

**Descrição:**
Ao tocar no botão "Galeria" na tela de adição de foto durante o onboarding, o app exibe o mesmo dialog de erro: **"Falha ao selecionar imagem. Tente novamente."** O picker de galeria abre mas a seleção da imagem falha ao tentar carregá-la no app.

**Correção esperada:**
Verificar o callback de retorno do picker de galeria. O erro pode estar relacionado à permissão `READ_MEDIA_IMAGES` (Android 13+) não sendo solicitada em runtime, ou a um problema na conversão do URI da imagem selecionada.

## AP-PRF-006 — Erro ao adicionar foto no Álbum pela galeria

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Alta |
| Tela | Álbum |
| Responsável | — |

**Descrição:**
Na tela de Álbum, ao tocar no botão `+` e selecionar uma imagem da galeria do celular, o mesmo dialog de erro é exibido: **"Falha ao selecionar imagem. Tente novamente."** O problema afeta tanto o upload pelo onboarding quanto pelo álbum, sugerindo falha no componente compartilhado de seleção de imagem.

## AP-PRF-007 — Nome do usuário sem fonte padrão nas configurações

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Média |
| Tela | Configurações / Conta |
| Responsável | — |

**Descrição:**
Na tela de configurações da conta, o nome do usuário exibido abaixo do avatar (ex: "Fabio Lino") não está usando a fonte **Plus Jakarta Sans**. O texto está renderizando com a fonte padrão do sistema.

**Correção esperada:**
Aplicar o token tipográfico correto (`heading-md`, Plus Jakarta Sans, weight 600) no componente de exibição do nome do usuário na tela de configurações.

## AP-PRF-008 — Renomear seção "Social" para nomenclatura mais clara

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Baixa |
| Tela | Configurações / Conta |
| Responsável | — |

**Descrição:**
O item de menu "Social" na tela de configurações não comunica claramente o que contém. O nome é genérico e pode confundir o usuário sobre o que encontrará ao acessar.

| Campo | Valor |
|-------|-------|
| Status | Aberto |

| Tela | Estatísticas |

## AP-PRF-010 — Faixa sólida sobre foto do perfil público deve ser substituída por gradiente

| Campo | Valor |
|-------|-------|
| Status | Aberto |
| Prioridade | Média |
| Tela | Perfil público (visualização) |
| Responsável | — |

**Descrição:**
Na tela de visualização do perfil público, o nome e @username do usuário são exibidos sobre uma faixa/banner de cor sólida na parte inferior da foto. Esse tratamento é visualmente pesado e cobre parte da imagem de forma abrupta.

**Correção esperada:**
Substituir a faixa sólida por um **gradiente linear** `transparent → rgba(0,0,0,0.7)` aplicado na base da foto (últimos 30-40% de altura). O texto do nome deve ficar sobre o gradiente com cor branca para garantir legibilidade.
