# Casos de Teste — Matching

Testes para validar o algoritmo e a experiência de matching entre usuários.

<Callout type="info">
**Pré-condições gerais:** Usuário autenticado com perfil completo e pelo menos 1 foto. Localização ativa. Preferências de busca configuradas.
</Callout>

## Resumo dos Casos

| ID | Cenário | Severidade |
|----|---------|------------|
| TC-MATCH-001 | Match mútuo entre dois usuários | Crítica |
| TC-MATCH-002 | Notificação de novo match | Alta |
| TC-MATCH-003 | Match com filtros de preferência | Alta |
| TC-MATCH-004 | Desfazer match (unmatch) | Média |
| TC-MATCH-005 | Lista de matches vazia | Baixa |
| TC-MATCH-006 | Match com usuário bloqueado | Alta |

## TC-MATCH-001: Match mútuo entre dois usuários

**Severidade:** Crítica

**Pré-condições:**
- Usuário A e Usuário B com perfis completos
- Nenhum match prévio entre eles

<Steps>
  <Step title="Usuário A curte Usuário B">
    Usuário A faz swipe right no perfil do Usuário B.
  </Step>
  <Step title="Usuário B curte Usuário A">
    Usuário B faz swipe right no perfil do Usuário A.
  </Step>
  <Step title="Verificar tela de match">
    A tela de "It's a Match!" deve ser exibida para ambos os usuários.
  </Step>
  <Step title="Verificar lista de matches">
    O novo match deve aparecer na lista de matches de ambos os usuários.
  </Step>
</Steps>

**Resultado esperado:** Match é criado com sucesso. Ambos os usuários veem a animação de match e o perfil aparece na lista de matches. O chat entre eles fica disponível.

---

## TC-MATCH-002: Notificação de novo match

**Severidade:** Alta

**Pré-condições:**
- Usuário A já curtiu Usuário B
- Usuário B tem notificações push ativadas
- Usuário B está com o app em background

<Steps>
  <Step title="Usuário B curte Usuário A">
    Usuário B faz swipe right no perfil do Usuário A (gerando o match).
  </Step>
  <Step title="Verificar push notification">
    Usuário A deve receber uma notificação push informando o novo match.
  </Step>
  <Step title="Abrir notificação">
    Ao tocar na notificação, o app deve abrir diretamente na tela do match ou na conversa.
  </Step>
</Steps>

**Resultado esperado:** Notificação push é enviada em até 5 segundos. Ao tocar, o app abre no contexto correto (tela de match ou chat).

---

## TC-MATCH-003: Match com filtros de preferência

**Severidade:** Alta

**Pré-condições:**
- Usuário A com filtros configurados: idade 25-35, distância até 50km, gênero feminino
- Usuário B dentro dos critérios (mulher, 28 anos, 30km de distância)
- Usuário C fora dos critérios (mulher, 40 anos, 100km de distância)

<Steps>
  <Step title="Usuário A navega nos perfis">
    Abrir a tela de descoberta e verificar os perfis sugeridos.
  </Step>
  <Step title="Verificar Usuário B aparece">
    O perfil do Usuário B deve estar disponível para swipe.
  </Step>
  <Step title="Verificar Usuário C não aparece">
    O perfil do Usuário C não deve ser exibido na fila de perfis.
  </Step>
</Steps>

**Resultado esperado:** Apenas perfis que atendem aos filtros de preferência são exibidos. Usuário C é filtrado corretamente.

---

## TC-MATCH-004: Desfazer match (unmatch)

**Severidade:** Média

**Pré-condições:**
- Usuário A e Usuário B já possuem um match ativo
- Conversa existente entre eles

<Steps>
  <Step title="Abrir perfil do match">
    Usuário A abre o perfil do Usuário B na lista de matches.
  </Step>
  <Step title="Selecionar opção de unmatch">
    Tocar no menu de opções e selecionar "Desfazer match".
  </Step>
  <Step title="Confirmar ação">
    Confirmar na modal de confirmação.
  </Step>
  <Step title="Verificar remoção">
    O Usuário B deve ser removido da lista de matches do Usuário A.
  </Step>
  <Step title="Verificar lado do Usuário B">
    No app do Usuário B, o Usuário A também deve desaparecer da lista de matches e o histórico de chat deve ser removido.
  </Step>
</Steps>

**Resultado esperado:** Match é desfeito para ambos os lados. Conversa e histórico são removidos. O perfil pode voltar a aparecer na fila de descoberta futuramente.

---

## TC-MATCH-005: Lista de matches vazia

**Severidade:** Baixa

**Pré-condições:**
- Usuário novo sem nenhum match

<Steps>
  <Step title="Abrir tela de matches">
    Navegar até a aba de matches/conversas.
  </Step>
  <Step title="Verificar estado vazio">
    A tela deve exibir um estado vazio com mensagem orientativa.
  </Step>
</Steps>

**Resultado esperado:** Mensagem como "Você ainda não tem matches. Continue explorando!" é exibida com ilustração. Sem erros ou tela em branco.

## Cenários Negativos / Edge Cases

### TC-MATCH-006: Match com usuário bloqueado

**Severidade:** Alta

**Pré-condições:**
- Usuário A bloqueou Usuário D previamente
- Usuário D tem perfil ativo

<Steps>
  <Step title="Usuário D curte Usuário A">
    Usuário D faz swipe right no perfil do Usuário A.
  </Step>
  <Step title="Verificar que Usuário A não vê Usuário D">
    O perfil do Usuário D não deve aparecer na fila de descoberta do Usuário A.
  </Step>
  <Step title="Verificar que nenhum match é criado">
    Mesmo que ambos curtam, nenhum match deve ser gerado entre usuários com bloqueio ativo.
  </Step>
</Steps>

**Resultado esperado:** Usuários bloqueados nunca geram match. O perfil bloqueado não aparece na fila de descoberta. Nenhuma notificação é enviada.
