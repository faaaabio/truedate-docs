# Casos de Teste — Likes

Testes para validar o sistema de likes, super likes e a tela "Quem curtiu você".

<Callout type="info">
**Pré-condições gerais:** Usuário autenticado com perfil completo. Conexão com internet ativa. Contadores de likes diários resetados.
</Callout>

## Resumo dos Casos

| ID | Cenário | Severidade |
|----|---------|------------|
| TC-LIKE-001 | Enviar like em perfil | Crítica |
| TC-LIKE-002 | Receber notificação de like | Alta |
| TC-LIKE-003 | Visualizar "Quem curtiu você" | Alta |
| TC-LIKE-004 | Super Like | Alta |
| TC-LIKE-005 | Limite diário de likes (free) | Média |
| TC-LIKE-006 | Like em perfil já curtido | Baixa |

---

## TC-LIKE-001: Enviar like em perfil

**Severidade:** Crítica

**Pré-condições:**
- Usuário A na tela de descoberta
- Perfil do Usuário B disponível na fila

<Steps>
  <Step title="Visualizar perfil do Usuário B">
    O card exibe foto, nome, idade e informações básicas.
  </Step>
  <Step title="Curtir o perfil">
    Fazer swipe right ou tocar no botão de like.
  </Step>
  <Step title="Verificar feedback visual">
    Animação de like é exibida. Contador de likes diários é decrementado (se usuário free).
  </Step>
  <Step title="Verificar no backend">
    Registro de like criado com timestamp, ID do remetente e destinatário.
  </Step>
</Steps>

**Resultado esperado:** Like é registrado no servidor. Se o Usuário B já curtiu o Usuário A, um match é gerado automaticamente. Contador de likes é atualizado.

---

## TC-LIKE-002: Receber notificação de like

**Severidade:** Alta

**Pré-condições:**
- Usuário B tem notificações push ativadas
- Usuário B é Premium (pode ver quem curtiu)

<Steps>
  <Step title="Usuário A curte Usuário B">
    Usuário A faz like no perfil do Usuário B.
  </Step>
  <Step title="Verificar notificação push">
    Usuário B recebe push notification: "Alguém curtiu você!" (ou mensagem similar).
  </Step>
  <Step title="Verificar badge na aba">
    O ícone da aba de likes deve mostrar um badge com o número de likes pendentes.
  </Step>
  <Step title="Abrir notificação">
    Ao tocar na notificação, o app deve navegar para a tela de likes recebidos.
  </Step>
</Steps>

**Resultado esperado:** Notificação é enviada em até 5 segundos. Badge é atualizado em tempo real. Navegação pela notificação leva ao contexto correto.

---

## TC-LIKE-003: Visualizar "Quem curtiu você"

**Severidade:** Alta

**Pré-condições:**
- Usuário B (Premium) tem 3+ likes pendentes
- Usuário C (Free) tem 3+ likes pendentes

<Tabs>
  <Tab title="Usuário Premium">
    <Steps>
      <Step title="Abrir tela de likes">
        Navegar até a seção "Quem curtiu você".
      </Step>
      <Step title="Verificar lista completa">
        Todos os perfis que curtiram devem ser exibidos com foto e informações visíveis.
      </Step>
      <Step title="Interagir com um perfil">
        Tocar em um perfil da lista para ver detalhes e decidir curtir de volta ou rejeitar.
      </Step>
    </Steps>

    **Resultado esperado:** Usuário Premium vê todos os perfis que o curtiram, com fotos nítidas e informações completas. Pode curtir de volta diretamente da lista.
  </Tab>
  <Tab title="Usuário Free">
    <Steps>
      <Step title="Abrir tela de likes">
        Navegar até a seção "Quem curtiu você".
      </Step>
      <Step title="Verificar perfis borrados">
        Os perfis devem ser exibidos com fotos borradas/desfocadas.
      </Step>
      <Step title="Verificar CTA de upgrade">
        Um botão de upgrade para Premium deve ser exibido de forma destacada.
      </Step>
    </Steps>

    **Resultado esperado:** Usuário Free vê a quantidade de likes mas com fotos borradas. CTA de upgrade é claro e funcional. Ao tocar no CTA, a tela de planos é aberta.
  </Tab>
</Tabs>

## TC-LIKE-004: Super Like

**Severidade:** Alta

**Pré-condições:**
- Usuário A tem super likes disponíveis (1/dia para free, 5/dia para premium)
- Perfil do Usuário B na tela de descoberta

<Steps>
  <Step title="Ativar Super Like">
    Fazer swipe up no card do perfil ou tocar no botão de estrela/super like.
  </Step>
  <Step title="Verificar animação diferenciada">
    Animação especial de super like (estrela azul, efeito brilhante) deve ser exibida.
  </Step>
  <Step title="Verificar notificação ao destinatário">
    Usuário B recebe notificação diferenciada: "Você recebeu um Super Like!"
  </Step>
  <Step title="Verificar destaque no perfil">
    Quando o Usuário B vir o perfil do Usuário A na fila, deve haver indicação visual de que foi um super like (borda azul, estrela).
  </Step>
  <Step title="Verificar contador">
    O contador de super likes disponíveis do Usuário A deve ser decrementado.
  </Step>
</Steps>

**Resultado esperado:** Super like é registrado com diferenciação. Usuário B recebe notificação especial. O perfil do Usuário A aparece destacado na fila do Usuário B. Contador atualizado.

## TC-LIKE-005: Limite diário de likes (free tier)

**Severidade:** Média

**Pré-condições:**
- Usuário C (plano Free) com limite de likes quase esgotado (ex: 1 like restante)

<Steps>
  <Step title="Usar último like disponível">
    Fazer swipe right para usar o último like do dia.
  </Step>
  <Step title="Tentar dar mais um like">
    Fazer swipe right em outro perfil.
  </Step>
  <Step title="Verificar bloqueio">
    O app deve exibir modal informando que o limite diário foi atingido.
  </Step>
  <Step title="Verificar CTA de upgrade">
    O modal deve incluir opção de upgrade para likes ilimitados e um timer mostrando quando os likes serão resetados.
  </Step>
  <Step title="Verificar reset">
    Após o período de reset (00:00 local), o contador deve ser restaurado ao valor inicial.
  </Step>
</Steps>

**Resultado esperado:** Swipe é bloqueado após limite. Modal informativo com timer e CTA de upgrade. Reset automático no horário correto. Usuário Premium nunca vê esta limitação.

## Cenários Negativos / Edge Cases

### TC-LIKE-006: Like em perfil já curtido

**Severidade:** Baixa

**Pré-condições:**
- Usuário A já curtiu Usuário B anteriormente
- Nenhum match foi gerado (Usuário B não curtiu de volta)

<Steps>
  <Step title="Verificar que o perfil não reaparece">
    O perfil do Usuário B não deve aparecer novamente na fila de descoberta do Usuário A.
  </Step>
  <Step title="Tentativa via deep link ou manipulação">
    Caso o perfil seja acessado por outro caminho, o botão de like deve estar desabilitado ou o sistema deve impedir o like duplicado.
  </Step>
</Steps>

**Resultado esperado:** O sistema impede likes duplicados. Nenhum registro duplicado é criado no banco. O perfil já curtido é excluído da fila de descoberta.
