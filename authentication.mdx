## Visão geral

A API do TrueDate usa autenticação via **Bearer Token** (JWT). Todos os endpoints requerem um token válido no header `Authorization`, exceto os de registro e login.

<Callout kind="info">
A URL base da API é `https://api.truedate.app/v1`. Todas as requisições devem usar HTTPS.
</Callout>

## Obtendo um token

Faça login com e-mail e senha para receber um `access_token` e um `refresh_token`:

```bash
curl -X POST https://api.truedate.app/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "user@example.com", "password": "sua_senha"}'
```

**Resposta:**

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4...",
  "expires_in": 3600,
  "token_type": "Bearer"
}
```

## Usando o token

Inclua o `access_token` no header `Authorization` em todas as requisições autenticadas:

```bash
curl https://api.truedate.app/v1/matches \
  -H "Authorization: Bearer SEU_ACCESS_TOKEN"
```

<ParamField header="Authorization" param-type="string" required="true">
  `Bearer <access_token>` — Token JWT obtido via login.
</ParamField>

## Renovando o token

O `access_token` expira em 1 hora. Use o `refresh_token` para obter um novo sem precisar fazer login novamente:

```bash
curl -X POST https://api.truedate.app/v1/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refresh_token": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4..."}'
```

<Callout kind="tip">
O `refresh_token` tem validade de 30 dias. Implemente a renovação automática no cliente para evitar interrupções na sessão do usuário.
</Callout>

## Autenticação social

O TrueDate também suporta login via OAuth2 com Google e Apple:

<Tabs>
  <Tab title="Google">
    ```bash
    curl -X POST https://api.truedate.app/v1/auth/google \
      -H "Content-Type: application/json" \
      -d '{"id_token": "TOKEN_DO_GOOGLE"}'
    ```
  </Tab>
  <Tab title="Apple">
    ```bash
    curl -X POST https://api.truedate.app/v1/auth/apple \
      -H "Content-Type: application/json" \
      -d '{"identity_token": "TOKEN_DA_APPLE", "user": "user_id_apple"}'
    ```
  </Tab>
</Tabs>

Ambos retornam a mesma estrutura de `access_token` e `refresh_token` do login tradicional.

## Erros comuns

<Expandable title="401 Unauthorized" default-open="false">
O token está ausente, inválido ou expirado. Verifique:

- O header `Authorization` está presente e formatado como `Bearer <token>`
- O `access_token` não expirou (válido por 1 hora)
- Se expirou, use o `refresh_token` para renovar
</Expandable>

<Expandable title="403 Forbidden" default-open="false">
O token é válido, mas o usuário não tem permissão para o recurso solicitado. Exemplos:

- Tentar acessar o chat de um match que não existe
- Tentar enviar mensagem para um usuário que deu unmatch
- Funcionalidade exclusiva de Premium acessada com conta Free
</Expandable>

<Expandable title="429 Too Many Requests" default-open="false">
Limite de requisições atingido. A API aceita até 100 requisições por minuto por token. Implemente retry com backoff exponencial no cliente.
</Expandable>
